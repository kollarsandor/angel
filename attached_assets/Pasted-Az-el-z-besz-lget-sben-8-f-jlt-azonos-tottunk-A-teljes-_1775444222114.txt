## Ami NEM probléma

### `src/verification/mgt.lisp` — `attention-mask-from-tokens`

Ez **nem** attention mechanizmus. Az `attention-mask-from-tokens` egy standard NLP tokenizáló utility, ami bináris maszkot generál (0 = padding, 1 = valós token):

```lisp
(defun attention-mask-from-tokens (tokens)
  (if (atom tokens) nil
    (cons (if (equal (car tokens) *special-pad*) 0 1)
          (attention-mask-from-tokens (cdr tokens)))))
``` [9](#5-8) 

A `make-position-ids` is csak szekvenciális indexeket generál `[0, 1, 2, ...]`, nem sinusoidális pozíciókódolást. A `token-type-ids` szegmens-azonosító. Ezek **minden NLP rendszerben** szükségesek, architektúrától függetlenül. Megtarthatók. [10](#5-9)

(defun make-position-ids (n start)
  (declare (xargs :guard (and (natp n) (natp start))
                  :measure (nfix n)))
  (if (zp n)
      nil
    (cons start (make-position-ids (- n 1) (+ start 1)))))
